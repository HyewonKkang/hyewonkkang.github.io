[TIL] 러닝타입스크립트 - 8. 클래스

Posted by Hyewon Kang on Wed, May 3, 2023

⚠️ 러닝 타입스크립트 책의 모든 내용을 담지 않으며 주관적인 기준으로 필요한 내용만 작성하고, 부족한 내용에 대해 추가 개념을 정리합니다.

클래스 속성

  • 함수 속성

    • 메서드 접근 방식은 함수를 클래스 프로토타입에 할당하므로 모든 클래스 인스턴스는 동일한 함수 정의를 사용한다.

      1class WithMethod {
2    myMethod() {}
3}
4
5new WithMethod().myMethod = new WithMethod().myMethod; // true

    • 값이 함수인 속성을 선언하는 방식은 클래스의 인스턴스당 새로운 함수가 생성되며, 항상 클래스 인스턴스를 가리켜야 하는 화살표 함수에서 this 스코프를 사용하면 클래스 인스턴스당 새로운 함수를 생성하는 시간과 메모리 비용 측면에서 유용할 수 있다.

      1class WithProperty {
2    myProperty: () => {};
3}
4
5new WithProperty().myProperty = new WithProperty().myProperty; // false

  • 초기화 검사

    • 엄격한 컴파일러 설정이 활성화된 상태(strictPropertyInitialization)에서 TS는 undefined 타입으로 선언된 각 속성이 생성자에게 할당되었는지 확인한다.

       1class WithValue {
 2    immediate = 0; // OK
 3    later: number; // OK(constructor에서 할당)
 4    mayBeUndefined: number | undefined; // Ok (undefined가 되는 것을 허용함)
 5    unused: number; // ERROR:  Property 'unused' has no initializer and is not definitely assigned in the constructor.
 6
 7    constructor() {
 8        this.later = 1;
 9    }
10}

    • 아래 예제는 엄격한 초기화 검사가 수행되지 않으면 올바르게 컴파일되지만, 런타임 시 문제가 발생한다.

      1class MissingInitializer {
2    property: string;
3}
4
5new MissingInitializer().property.length; // TypeError: Cannot read property 'length' of undefined

    • 확실하게 할당된 속성

      • 엄격한 초기화 검사가 유용한 경우가 대부분이지만 클래스 생성자 다음에 클래스 속성을 의도적으로 할당하지 않는 경우가 있을 수도 있다. ⇒ 이름 뒤에 !를 추가해 검사를 비활성화하도록 한다.

        → 타입스크립트에 속성이 처음 사용되기 전에 undefined 값이 할당된다.

         1class ActivitiesQueue {
 2    pending!: string[]; // OK
 3
 4    initialize(pending: string[]) {
 5        this.pending = pending;
 6    }
 7
 8    next() {
 9        return this.pending.pop();
10    }
11}
12
13const activities = new ActivitiesQueue();
14activities.initialize(['eat', 'sleep', 'learn']);
15activities.next();

  • 읽기 전용 속성

    • 인터페이스와 마찬가지로 클래스도 선언된 속성 이름 앞에 readonly 키워드를 추가해 속성을 읽기전용으로 선언한다.
    • readonly로 선언된 속성은 선언된 위치 또는 생성자에서 초기값만 할당할 수 있다.

타입으로서의 클래스

💡 타입시스템에서의 클래스는 클래스 선언이 런타임 값(클래스 자체) 과 타입 애너테이션에서 사용할 수 있는 타입을 모두 생성한다. ⇒ 클래스는 타입이자 클래스 그 자체로 사용

1class Teacher {
2    sayHello() {
3        console.log('HI');
4    }
5}
6
7let teacher: Teacher;
8
9teacher = new Teacher();

 4}
 5

11    }

14        for (let i = 0; i < hours; i++) {

클래스 확장

  • 할당 가능성 확장

    → 파생된 인스턴스는 기본 클래스 또는 하위 클래스를 충족하는 데 사용할 수 있다.

     1class Lesson {
 2    subject: string;
 3
 4    constructor(subject: string) {
 5        this.subject = subject;
 6    }
 7}
 8
 9class OnlineLesson extends Lesson {
10    url: string;
11
12    constructor(subject: string, url: string) {
13        super(subject);
14        this.url = url;
15    }
16}
17
18let lesson: Lesson;
19lesson = new Lesson('coding'); // OK
20lesson = new OnlineLesson('coding', 'lesson.com'); // OK

추상 클래스

  • abstract를 이용해 하위 클래스가 해당 메서드를 제공할 것을 예상하고 기본 클래스를 구현

     1abstract class School {
 2    readonly name: string;
 3
 4    constructor(name: string) {
 5        this.name = name;
 6    }
 7
 8    abstract getStudentTypes(): string[];
 9}
10
11class Preschool extends School {
12    getStudentTypes(): string[] {
13        return ['preschooler'];
14    }
15}
16
17class Absense extends School {} // ERROR: Non-abstract class 'Absense' does not implement inherited abstract member 'getStudentTypes' from class 'School'.

멤버 접근성

  • 자바스크립트 → #으로 private 클래스 멤버임을 나타낸다.

  • 타입스크립트 → 타입 시스템에만 존재하는 클래스 메서드와 속성에 대해 조금 더 미묘한 프라이버시 정의 집합을 허용한다. 이러한 키워드는 순수하게 타입 시스템 내에 존재한다. 코드가 JS로 컴파일되면 다른 모든 타입 시스템 구문과 함께 키워드도 제거된다.

    • public(기본값): 모든 곳에서 누구나 접근 가능
    • protected: 클래스 내부 또는 하위 클래스에서만 접근 가능
    • private: 클래스 내부에서만 접근 가능

  • 접근성 제한자는 readonly와 함꼐 표시할 수 있다.

     1class TwoKeywords {
 2    private readonly name: string;
 3
 4    constructor() {
 5        this.name = 'Anne Sullivan';
 6    }
 7
 8    log() {
 9        console.log(this.name);
10    }
11}
12
13const two = new TwoKeywords();
14
15two.name = 'Savitribai Phule'; // Error

  • 정적 필드 제한자 static

    • 자바스크립트는 static 키워드를 이용해 클래스 자체에서 멤버를 선언한다.
    • 타입스크립트는 static 키워드를 단독으로 사용하거나 readonly와 접근성 키워드를 함께 사용할 수 있도록 지원한다.
     1class Question {
 2    protected static readonly answer: 'bash';
 3    protected static readonly prompt = "What's an ogre's favorite programming language?";
 4
 5    guess(getAnswer: (prompt: string) => string) {
 6        const answer = getAnswer(Question.prompt);
 7
 8        if (answer === Question.answer) {
 9            console.log('You got it!');
10        } else {
11            console.log('Try again...');
12        }
13    }
14}
15
16Question.answer; // ERROR: Property 'answer' is protected and only accessible within class 'Question' and its subclasses.
