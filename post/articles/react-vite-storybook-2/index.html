Vite + React + TypeScript 환경에 Storybook v7.0 도입하기 (2)

Posted by Hyewon Kang on Thu, May 11, 2023

개요

스토리북의 기본 사용방법에 대해 Vite + React + TypeScript 환경에 Storybook v7.0 도입하기 (1) 에서 학습했다. 이번에는 Storybook을 더 잘 사용하기 위한 몇 가지 방법들을 활용해보려고 한다.

1. Code panel 추가하기

현재 기본적으로 Controls, Actions, Interaction, Knobs 패널이 존재한다. 여기에 컴포넌트의 코드를 바로 확인할 수 있도록 Code panel을 추가하려고 한다.

간단하게 @storybook/addon-storysource를 설치하고, addons에 규칙을 추가해주면 된다.

1yarn add @storybook/addon-storysource --dev

 1// .storybook/main.js
 2
 3module.exports = {
 4  addons: [
 5    '@storybook/addon-links',
 6    '@storybook/addon-essentials',
 7    '@storybook/addon-mdx-gfm',
 8    '@storybook/addon-knobs',
 9    **'@storybook/addon-storysource',** // 추가
10  ],
11	...
12};

그러면 하단에 Code 패널이 생성된다.

2. 디자인시스템 관련 페이지 작성하기

Storybook 자체가 프론트엔드 개발의 문서화를 지원한다. 그래서 컴포넌트 테스트용 목적 외에도 프로젝트를 소개하는 페이지를 추가하고자 한다.

우선 완성된 결과물은 다음과 같다. 좌측에 GETTING STARTED와 COMPONENTS 그룹을 생성했고, 프로젝트에 대한 소개인 Intro 페이지, 프로젝트에서 사용되는 icons를 정의한 Icons 페이지, 컬러토큰을 정의한 Palette 페이지, 폰트 관련 서식을 정의한 Typography 페이지를 추가로 구성했다.

2-1) Naming components and hierarchy + Grouping + Sorting

다음과 같이 파일을 그룹핑하는 방법부터 알아보자. stories.(tsx|mdx) 파일을 생성한 후, title에서 이를 설정할 수 있다.

Button 컴포넌트를 예로 들면, 아래와 같이 title에 / separator를 이용하면 그룹끼리 묶일 수 있다.

1// src/components/Button/Button.stories.tsx
2
3import Button from './Button';
4
5export default {
6    title: 'Components/Button',
7    component: Button,
8};

추가적으로, 정렬하는 방법에 대해 알아보자.

좌측 메뉴의 순서는 기본적으로 추가된 페이지 순으로 정렬되는데, 이러한 규칙을 알파벳 오름차순으로 정렬시키려면, 다음과 같이 parameters options의 storySort를 이용한다.

method에 ‘alphabetical’을 추가하면 알파벳 순서로 스토리가 정렬된다. 이밖에도, 특정 페이지를 맨 위나 맨 아래로 배치하고 싶은 경우가 있다면, order 규칙을 적용할 수 있다. 이는 사용자 임의로 순서를 결정짓는데, 아래와 같은 규칙을 통해 GETTING STARTED 그룹의 순서가 알파벳 규칙을 따르지 않는 것을 확인할 수 있다.

 1export default {
 2    parameters: {
 3        options: {
 4            storySort: {
 5                method: 'alphabetical',
 6                order: [
 7                    'Getting Started',
 8                    ['Intro', 'Icons', 'Palette', 'Typography'],
 9                    'Components',
10                ],
11            },
12        },
13    },
14};

2-2) Introduction 페이지 작성하기

이제 본격적으로 각 페이지를 작성해보자.

Getting Started 하위 페이지 생성을 위해 src/stories 경로에 각 파일들을 작성했고, 컴포넌트를 정의하는 용도가 아닌 markdown 문서를 작성하기 위한 페이지이기 때문에 확장자는 mdx로 작성했다.

다음 그림은 누가봐도 마크다운 문법으로 작성된 페이지다. Storybook에서 제공하는 API 가운데 Meta 를 이용해 title을 정의한다. 그리고 해당 페이지에 대한 내용은 다들 익숙한 md 문법으로 작성하면 된다!

 1// src/stories/intro.stories.mdx
 2
 3import React from 'react';
 4import { Meta } from '@storybook/blocks';
 5import IconButton from '../components/IconButton';
 6import { ThemeProvider } from 'styled-components';
 7import { theme } from '../styles/theme';
 8
 9<Meta title='Getting Started/Intro' />
10
11<section style={{ display: 'flex', flexDirection: 'column', justifyContent: 'center', alignItems: 'center', padding: '10px' }}>
12
13# Classmate Design System
14
15### Design foundations and components library
16
17<div style={{ padding: '20px' }}>
18  <ThemeProvider theme={theme}>
19    <IconButton
20      name='github'
21      onClick={() => window.open('https://github.com/Dev-TeamOne/classmate-frontend', '_blank')}
22    />
23  </ThemeProvider>
24</div>
25
26</section>
27

2-3) IconGallery 만들기

이 역시도 Storybook에서 기본 API를 제공한다. IconGallery를 통해 프로젝트에서 사용되는 아이콘들을 grid 형태로 진열시킨다. IconGallery의 children으로 IconItem들을 넣을 수 있다.

 1import React from 'react';
 2import { Meta, IconGallery, IconItem } from '@storybook/blocks';
 3import Icon from '../components/Icon';
 4import { icons } from '../components/Icon/utils';
 5
 6<Meta title='Getting Started/Icons' />
 7
 8# IconGallery
 9
10<IconGallery style={{ marginTop: '50px' }}>
11{icons.map((key) => (
12<IconItem name={key}>
13<Icon name={key} />
14</IconItem>
15))}
16</IconGallery>

2-4) ColorPalette 만들기

ColorPalette도 사용법이 간단하다. ColorPalette 컴포넌트의 children으로 나타내는 색상의 ColorItem을 작성하면된다.

  • title : string 표시할 색상의 이름
  • subtitle : string 색상에 대한 추가설명
  • colors : string[] | { [key: string]: string } 표시할 색상 목록 (key:value 형태)

예시의 grey와 같이 여러 색상으로 분리하려면 color에 순서대로 key:value를 추가해주면 된다.

1<ColorItem
2title='grey'
3subtitle='greyscale'
4colors={{ Grey1: '#888888', Grey2: '#BBBBBB', Grey3: '#D7D7D7' }}
5/>

2-5) Typeset 만들기

Typeset block은 프로젝트에서 쓰이는 폰트를 나타내는 목적의 문서다. Typeset 컴포넌트의 props로 fontSizes 배열과 fontWeight, fontFamily 를 적용할 수 있으며, 예시 문구는 sampleText로 전달한다.

  • fontSizes

 1import React from 'react';
 2import { Meta, Typeset } from '@storybook/blocks';
 3import { theme } from '../styles/theme';
 4
 5<Meta title='Getting Started/Typography' />
 6
 7# Base Typography
 8
 9**Font:** Noto Sans KR
10
11**Weights:** 400(light), 500(text), 700(bold, title)
12
13<Typeset fontSizes={[12, 14, 16, 18, 28, 32]} fontWeight={400} sampleText='Was he a beast if music could move him so?' />
14<Typeset fontSizes={[12, 14, 16, 18, 28, 32]} fontWeight={500} />
15<Typeset fontSizes={[12, 14, 16, 18, 28, 32]} fontWeight={700} />

이외에도 Storybook은 다양한 API를 지원한다. 조금 더 문서를 잘 작성하고 싶다면 storybook-docs-api 문서를 보면서 추가해보는 것을 추천한다.
